## Overview

This guide will walk you through cloning the repository, building the app, and logging in for the first time. By the end, you'll have the app running on an Android device or emulator with a test user account.

<Note>
  Before starting, ensure you've completed the [Installation](/installation) guide with Android Studio and Firebase configured.
</Note>

## Step 1: Clone the Repository

<Steps>
  <Step title="Clone from GitHub">
    Open your terminal and run:
    
    ```bash
    git clone https://github.com/gargmehul10/Sales-Management-App.git
    cd Sales-Management-App
    ```
  </Step>
  
  <Step title="Verify Project Structure">
    Ensure the project structure looks like this:
    
    ```
    Sales-Management-App/
    ├── app/
    │   ├── build.gradle
    │   ├── google-services.json  # Add your Firebase config here
    │   └── src/
    │       └── main/
    │           ├── AndroidManifest.xml
    │           ├── java/
    │           │   └── project/avishkar/salesmanagement/
    │           │       ├── Registration/
    │           │       │   ├── MainActivity.java
    │           │       │   ├── SignUpSalesManager.java
    │           │       │   └── SignUpSalesperson.java
    │           │       ├── ManagerMain.java
    │           │       ├── SalespersonMain.java
    │           │       ├── Chat/
    │           │       ├── Graph/
    │           │       ├── Leaderboard/
    │           │       └── InventoryItem.java
    │           └── res/
    ├── build.gradle
    ├── settings.gradle
    └── gradle.properties
    ```
  </Step>
  
  <Step title="Add Firebase Configuration">
    Copy your `google-services.json` file into the `app/` directory:
    
    ```bash
    cp ~/Downloads/google-services.json app/
    ```
    
    <Warning>
      The app will not build without a valid `google-services.json` file. See [Firebase Configuration](/installation#step-3-firebase-configuration) if you haven't set this up yet.
    </Warning>
  </Step>
</Steps>

## Step 2: Build the Project

<Steps>
  <Step title="Open in Android Studio">
    1. Launch Android Studio
    2. Click **Open an Existing Project**
    3. Navigate to the cloned directory and click **OK**
    4. Wait for Gradle sync to complete
  </Step>
  
  <Step title="Sync Gradle Files">
    If Gradle doesn't sync automatically:
    
    1. Click **File > Sync Project with Gradle Files**
    2. Wait for dependencies to download (first time may take 5-10 minutes)
    
    You should see output like:
    ```
    Gradle sync finished in 1m 23s
    ```
  </Step>
  
  <Step title="Build the Application">
    Build the app to verify everything works:
    
    ```bash
    # From terminal
    ./gradlew assembleDebug
    
    # Or in Android Studio
    # Build > Make Project (Ctrl+F9)
    ```
    
    Expected output:
    ```
    > Task :app:compileDebugJavaWithJavac
    > Task :app:mergeDebugResources
    > Task :app:processDebugGoogleServices
    > Task :app:packageDebug
    
    BUILD SUCCESSFUL in 1m 45s
    47 tasks executed
    ```
  </Step>
</Steps>

## Step 3: Set Up an Emulator or Device

You can run the app on an Android emulator or physical device:

<Tabs>
  <Tab title="Android Emulator">
    <Steps>
      <Step title="Create a Virtual Device">
        1. In Android Studio, click **Tools > AVD Manager**
        2. Click **Create Virtual Device**
        3. Select a device (e.g., Pixel 3)
        4. Choose a system image:
           - **Recommended**: Oreo (API 26) or higher
           - **Minimum**: Lollipop (API 22)
        5. Click **Finish**
      </Step>
      
      <Step title="Launch Emulator">
        In AVD Manager, click the play button next to your device.
        
        Wait for the emulator to boot completely (30-60 seconds).
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Physical Device">
    <Steps>
      <Step title="Enable Developer Options">
        On your Android device:
        
        1. Go to **Settings > About Phone**
        2. Tap **Build Number** 7 times
        3. You'll see "You are now a developer!"
      </Step>
      
      <Step title="Enable USB Debugging">
        1. Go to **Settings > Developer Options**
        2. Enable **USB Debugging**
        3. Connect device via USB
        4. Accept the debugging prompt on your device
      </Step>
      
      <Step title="Verify Connection">
        ```bash
        adb devices
        ```
        
        You should see:
        ```
        List of devices attached
        ABC123XYZ456    device
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Understanding User Roles

The app authenticates users and determines their role from Firebase:

<CodeGroup>
```java Manager Login Flow (MainActivity.java:142)
if (RadioButtonSelect(radioGroup_type.getCheckedRadioButtonId()).equals("Manager")) {
    databaseRef = FirebaseDatabase.getInstance().getReference("Manager");
    databaseRef.addListenerForSingleValueEvent(new ValueEventListener() {
        @Override
        public void onDataChange(@NonNull DataSnapshot dataSnapshot) {
            for (DataSnapshot snapshot : dataSnapshot.getChildren()) {
                SalesManager sm1 = snapshot.getValue(SalesManager.class);
                if (sm1.getEmail().equals(email)) {
                    SessionManager sm = new SessionManager(getApplicationContext());
                    sm.createLoginSession(snapshot.getKey(), "Manager");
                    goto_Manager();
                    break;
                }
            }
        }
    });
}
```

```java Salesperson Login Flow (MainActivity.java:174)
else {
    databaseRef = FirebaseDatabase.getInstance().getReference("Salesperson");
    databaseRef.addListenerForSingleValueEvent(new ValueEventListener() {
        @Override
        public void onDataChange(@NonNull DataSnapshot dataSnapshot) {
            for (DataSnapshot snapshot : dataSnapshot.getChildren()) {
                SalesPerson sp1 = snapshot.getValue(SalesPerson.class);
                if (sp1.getEmailId().equals(email)) {
                    SessionManager sm = new SessionManager(getApplicationContext());
                    sm.createLoginSession(snapshot.getKey(), "Salesperson");
                    goto_Salesperson();
                }
            }
        }
    });
}
```
</CodeGroup>

<Warning>
  Users must select the correct role (Manager or Salesperson) that matches their registration. If a manager selects "Salesperson" during login, authentication will fail with:
  
  "This Email is not registered as Salesperson!!"
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="App Crashes on Launch">
    **Problem**: App crashes immediately after launch
    
    **Common causes**:
    1. Missing or invalid `google-services.json`
    2. Firebase services not enabled
    
    **Solution**:
    ```bash
    # Check logcat for errors
    adb logcat | grep -i firebase
    
    # Look for:
    # "Default FirebaseApp is not initialized"
    # "google-services.json is missing"
    ```
    
    Verify Firebase configuration in [Installation Guide](/installation#step-3-firebase-configuration).
  </Accordion>
  
  <Accordion title="Authentication Failed">
    **Problem**: "Authentication failed" error on login
    
    **Solution**:
    1. Verify Email/Password authentication is enabled in Firebase Console
    2. Check that password is at least 6 characters
    3. Ensure correct role is selected (Manager vs Salesperson)
    4. Verify internet connection (app requires network access)
  </Accordion>
  
  <Accordion title="Data Not Appearing">
    **Problem**: Inventory or chat messages don't show up
    
    **Solution**:
    1. Check Firebase Realtime Database rules:
       ```json
       {
         "rules": {
           ".read": true,
           ".write": true
         }
       }
       ```
    2. Verify internet connectivity
    3. Check Firebase Console to see if data is being written
    4. Look for permission errors in logcat
  </Accordion>
  
  <Accordion title="Images Not Loading">
    **Problem**: Profile pictures or images don't display
    
    **Solution**:
    1. Verify Firebase Storage is set up and accessible
    2. Check storage rules allow read/write
    3. Grant storage permissions:
       ```xml AndroidManifest.xml:6
       <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
       <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
       ```
    4. Accept permission prompts when the app requests them
  </Accordion>
</AccordionGroup>
